\# PREMORTEM.md



\##  Premortem: Why This Project Might Fail



이 프로젝트가 1년 후 실패했다고 가정하고,

그 이유를 미리 분석한다.



\---



\##  1. 사용자가 실제로 행동하지 않는다



\### 문제

\- 추천은 보지만 실행하지 않음

\- 협상, 해지 귀찮음



\### 대응

\- 자동화 기능 강화

\- “원클릭 실행” 구조

\- 알림 + 리마인드 시스템



\---



\##  2. 데이터 확보 문제



\### 문제

\- 금융/지출 데이터 연동 어려움

\- API 제한



\### 대응

\- 초기에는 수동 입력 기반

\- 점진적 API 확장

\- MVP는 단순 구조로 시작



\---



\##  3. 차별성이 부족하다



\### 문제

\- 기존 가계부 앱과 유사



\### 대응

\- “절약 → 투자 자동 연결”에 집중

\- 기능 축소하고 핵심 강화



\---



\##  4. AI 필요성이 약하다



\### 문제

\- 단순 rule-based로도 가능



\### 대응

\- 개인 맞춤 추천 강화

\- 행동 유도 메시지 생성

\- 투자 전략 개인화



\---



\##  5. 투자 기능 규제 리스크



\### 문제

\- 투자 추천은 법적 제한 존재



\### 대응

\- 직접 투자 권유 대신

\- “교육 + 시뮬레이션 중심”

\- ETF 기반 일반 전략 사용



\---



\##  6. 사용자 유지율 낮음



\### 문제

\- 초기에만 사용하고 이탈



\### 대응

\- 지속적 가치 제공

\- 월간 리포트

\- 자산 성장 시각화



\---



\##  7. 너무 많은 기능



\### 문제

\- 복잡함 → 사용성 저하



\### 대응

\- MVP는 1\~2 기능 집중

\- 핵심: 절감 + 투자 연결



\---



\##  핵심 교훈



이 서비스의 실패는 대부분



&#x20;“사용자가 행동하지 않는 것”에서 시작된다



\---

우리는



\- 분석 서비스 X

\- 추천 서비스 X



“행동을 유도하는 실행 서비스”
